Overview

casmi is an opensource free software and is Java based library with OpenGL that is using in production or development of 2D/3D visualization systems. It can support programmers with simple and intuitive interfaces for their implementation. It can work on cross platform (GNU/Linux, Mac OS X, and Windows). casmi is sutable for visualization, because it originaly desinged for information visualization, especially with huge data set.

Features

casmi has 5 big features.

  1. Open and Free
  2. Java based and using OpenGL
  3. Cross platform
  4. Simple and Intuitive
  5. Suitable for Information Visualization

Open and Free

casmi is an opensource and free software under LGPL license. You can free to download and use it. You are available to download source codes.

Java based and using OpenGL

casmi takes over features of Java and OpenGL. casmi can write in Java which is an object oriented language. It has function of gabbage collection, so you don't have to think about memory release. You can use extention using JNI which enables fast calculation with native machine code instead of JVM.

Cross platform

casmi works on cross platform environments (GNU/Linux, Mac OS X and Windows). It can also has a function of binary export for app(Mac OS X) and exe (Windows). You can use any environments.

Simple and Intuitive

casmi is designed for supporting programmers and developers to implement visualization systems quickly and easily. casmi has several convinient functions for supporting network, database IO, and File IO. It can support HTTP for network, SQLite for database, CSV, XML, and JSON for file IO. It will support for MySQL. It will also has functions for animation and timeline which controls various scenes by time.

Suitable for Information Visualization

casmi is designed for information visualization, especially with huge data set. It has methods to draw OpenGL graphics which can be used for visualization. casmi has a function to support for database IO and file IO, by combination of these functions, you can develop systems for visualization quite easily. It will has a fuction to generate graphs. It will be added simulation fuctions, for example, physical simulation.

FAQ

What is casmi?
casmi is a Java-based library for visualization, data access, and data analysis.It designed for programmers to write their application easily and quickly.And it is opensource.
Why is casmi opensource software?
Contributers want casmi to be used widely all over the world.And they hope there are many kind contributers to grow up casmi.
What kind of license does casmi have?
casmi has LGPL version 3 license for opensource version.
How can I do when I want to involved in the casmi project?
Please contact us. https://github.com/casmi
Who had an idea of casmi?
Members of Xcoo, Inc. got up the idea of casmi. They are developing customized visualization software, so they thought it would be convinient when they had a good library for visualization software. They started development of casmi from June 2011.
What the difference between casmi and precursors?
casmi is designed for visualization and serious applications. For visualization, there are Processing development environment and OpenFrameworks. They are our precursors. Their main target are artists and beginners of programing. Our target is business man and programmers.
If we want to ask someone to develop systems using casmi, what should we do?
Please contact Xcoo, Inc. They can help you.
What is Xcoo, Inc. ?
Xcoo, Inc. is a startup company for research and development of visualization and interactive systems, located in Tokyo, Japan. They began their business on April 1st, 2011. Background of members of Xcoo, Inc includes virtual reality, computer science, and design. Please visit their website. http://www.xcoo.jp/

<plugin>
	<groupId>org.codehaus.mojo</groupId>
	<artifactId>osxappbundle-maven-plugin</artifactId>
	<version>1.0-alpha-1</version>
	<configuration>
		<mainClass>your.application.MainClass</mainClass>
	</configuration>
	<executions>
		<execution>
			<phase>package</phase>
			<goals>
				<goal>bundle</goal>
			</goals>
		</execution>
	</executions>
</plugin>
